# Contributing to Trusted Validation Patterns

Thank you for your interest in contributing!

## How to Contribute

### Reporting Issues
- Use GitHub Issues for bugs, unclear documentation, 
  or suggestions
- Be as specific as possible

### Suggesting Changes
- Open an Issue before submitting a Pull Request
- Describe what you want to change and why

### Pull Requests
- Fork the repository
- Make your changes
- Submit a Pull Request with a clear description

## Guidelines
- Keep it simple and clear – this project values 
  simplicity over complexity
- Changes to existing patterns require strong 
  justification
- New extension patterns are welcome but must follow 
  the existing pattern structure

## Questions
Open an Issue and tag it with "question".
